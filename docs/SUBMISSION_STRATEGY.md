# Submission Strategy

## Target

Galuxium Nexus V2: build profitable SaaS platforms that real users will pay for.

Primary page:

```text
https://galuxium-nexus-v2.devpost.com/
```

## Winning Angle

Most AI hackathon projects are chatbots or content tools. Sentinel SaaS is framed as enterprise infrastructure for the AI coding wave:

> AI coding is making teams ship faster. Sentinel SaaS makes sure they do not ship dangerous releases faster.

## Judge Psychology

Judges should immediately see:

- A real market pain: AI-generated code creates release risk.
- A working product: web UI plus tested agent engine.
- Enterprise seriousness: audit trail, human approval, governance, transparent scoring.
- SaaS viability: pricing tiers, usage-based metering, target buyer.
- Demo completeness: a clean 2-5 minute workflow with no fluff.

## Required Devpost Assets

- Hosted demo URL
- Public GitHub repository
- 2-5 minute video
- Clear project story
- Monetization explanation
- Screenshots or gallery image

## Submission Checklist

- [ ] Deploy `web/` on Vercel, Netlify, GitHub Pages, or another public host.
- [ ] Push the complete repo to GitHub.
- [ ] Record demo using `docs/VIDEO_SCRIPT.md`.
- [ ] Paste Devpost fields from `docs/DEVPOST_COPY.md`.
- [ ] Upload one clean dashboard screenshot.
- [ ] Include the GitHub repository link and hosted demo link.

## Honest Probability

If submitted as only a repo and local script: 10-15%.

If submitted with a polished hosted dashboard, complete GitHub repo, 3-minute video, and clear SaaS pricing: 35-50%.

If the demo also includes a real GitHub PR webhook or deployment proof before final submission: 50-60%.

The top risk is not the idea. The top risk is whether the final submission looks like a production SaaS, not a local proof of concept.
