# Submission Strategy

## Primary Track

Track 3: UiPath Test Cloud.

## Awards To Attack

1. Best of UiPath Test Cloud
2. Best Demo / Presentation
3. Best Cross-Platform Integration
4. Best Product Feedback
5. Coding-agent bonus

## Judge Psychology

Judges see many workflow demos. Most will be simple "agent fills a form" projects. Sentinel QA Agent should feel different:

- It solves a painful enterprise trust problem.
- It shows failure, not just success.
- It makes UiPath the control plane.
- It has security/regression depth.
- It produces artifacts an enterprise buyer understands: test plan, risk score, failed evidence, release gate, approval queue.

## Winning Narrative

"AI can build and operate workflows, but enterprises cannot release what they cannot test. Sentinel QA Agent uses UiPath Test Cloud to turn risky changes into automated tests, evidence, and human approval gates."

## Demo Must Show

1. Risky AI refund workflow input.
2. Risk classifier finds payment/auth/agent/webhook risk.
3. Test Cloud plan is generated.
4. One simulated Test Cloud case fails.
5. Release is blocked.
6. Human review task is created.
7. Dashboard shows traceability.
8. UiPath integration files prove platform path.

## What Not To Do

- Do not pitch as "AI test generator" only.
- Do not overfocus on local Python.
- Do not hide the failure simulation; the failure is the story.
- Do not claim real UiPath execution until connected.
- Do not make it sound like cybersecurity scanning only; it is release governance through testing.
