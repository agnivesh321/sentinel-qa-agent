# AlgoFest Submission Strategy

## Track

Primary: **Artificial Intelligence & Machine Learning**

Secondary angle: **Open Innovation**

Optional story angle: **FinTech Innovations**, because the demo protects an AI-assisted invoice refund workflow.

## Positioning

Lead with this:

> Sentinel Release Guard is an AI release-risk agent for software teams. It reads risky changes, generates security regression gates, simulates CI evidence, and blocks unsafe releases before production.

Avoid positioning it as only a test generator. The winning angle is release safety and governance for AI-speed engineering teams.

## Judge Psychology

Judges will see many projects that are dashboards, chatbots, and wrappers around APIs. Sentinel should feel different because it:

- makes a concrete decision,
- shows evidence,
- blocks a risky action,
- has a visible dashboard,
- includes tests and benchmark output,
- can be explained in under two minutes.

## Highest-Value Demo Path

1. Show the problem: AI code and AI workflows can ship dangerous changes faster than humans can review.
2. Run the demo command.
3. Open the dashboard.
4. Point to risk score `96/100`.
5. Show blocked decision.
6. Show generated release-gate test cases.
7. Show failed simulated evidence.
8. Show human review queue and GitHub PR comment.
9. Close with: this is not a chatbot; it is a release-control agent.
