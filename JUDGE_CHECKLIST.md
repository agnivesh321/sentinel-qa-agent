# Judge Checklist

Use this to audit the submission before final upload.

## Required

- Public GitHub repo
- Open-source license
- Hosted URL
- Demo video, 5 minutes max
- Slide deck
- Clear setup instructions
- Track selected: UiPath Test Cloud

## Must Be Visible In Video

- UiPath Test Cloud positioning
- Risky AI refund workflow input
- Generated test cases
- Failed test evidence
- Blocked release decision
- Human approval task
- Integration payloads for UiPath

## Avoid Weaknesses

- Do not say "simulated" too often in the main pitch. Say "local demo mode" and immediately show the UiPath handoff payloads.
- Do not pitch broad QA automation. Pitch release trust for AI-built workflows.
- Do not spend video time on code internals. Show business risk, test cases, failure, release gate.
